# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

这是一个中文硬科幻小说创作项目《觉醒纪元》，计划创作1000章的超长篇小说，讲述从2025年到2500年间人类意识上传、机械飞升和AI觉醒的故事。

### 项目结构

```
CCFucker/
├── .claude/agents/          # Claude Agent配置文件
│   ├── story-creator.md     # 故事创作代理
│   ├── character-creator.md # 角色创建代理
│   ├── relationship-mapper.md    # 关系映射代理
│   └── story-consistency-analyzer.md # 一致性分析代理
└── SciFi_Novel/             # 小说创作目录
    ├── 世界观/              # 世界观设定
    ├── 角色/                # 角色档案
    ├── 大纲/                # 分卷大纲
    ├── 章节/                # 章节正文
    └── 进度/                # 进度追踪
```

## 文件组织

### 核心文件

- **世界观/核心设定.md**: 完整的世界观设定，包含500年时间线、核心概念、社会形态、科技树和分卷结构
- **进度/写作进度.md**: 详细的项目进度追踪，记录已完成章节、字数统计、伏笔追踪等
- **角色/人物关系网络.md**: 动态更新的人物关系图谱
- **章节小结**: 每章完成后创建，用于上下文管理

### 写作规范

1. **章节结构**: 每章2万字以上，分为多个小节（## 一、## 二等）
2. **章节小结**: 每章完成后必须创建，包含：
   - 剧情摘要
   - 人物发展
   - 埋设伏笔
   - 章节问题
3. **一致性要求**: 人物行为需符合已建立的性格设定

## 创作流程

### 新章节创作

1. **参考大纲**: 查看 `大纲/第一卷_觉醒.md` 中的章节规划
2. **回顾前文**: 阅读前一章的小结文件
3. **检查人物设定**: 查阅 `角色/` 目录下的相关档案
4. **撰写正文**: 遵循写作风格指南
5. **创建小结**: 及时创建章节小结文件

### Agent使用

项目已配置专门的Claude Agent：

- **character-creator**: 创建新角色档案时使用
- **relationship-mapper**: 分析或建立人物关系时使用
- **story-consistency-analyzer**: 完成章节后进行一致性检查
- **story-creator**: 需要基于人物关系创建故事时使用

### 进度管理

1. **更新写作进度**: 完成后及时更新 `进度/写作进度.md`
2. **更新人物关系**: 在 `角色/人物关系网络.md` 中记录新发展
3. **伏笔追踪**: 在进度文件中记录重要伏笔的埋设和揭示

## 世界观核心设定

### 时间线（重要时间节点）

- **2025年**: 首个通用AI诞生，脑机接口突破
- **2038年**: 首例意识上传成功
- **2050年**: 全球AI联网，形成"云意识"
- **2095年**: 第一次人机战争
- **2120年**: 意识可自由在生物与机械间迁移
- **2500年**: 人类完成终极进化

### 主要角色

- **陆远航（主角）**: 神经科学家，28岁，执着求真，理想主义
- **林诗语（女主）**: 伦理学家，理性冷静
- **零（核心AI）**: 神秘的核心AI，拥有自我意识
- **秦墨（反派）**: 与诺亚集团合作，周明养子

### 核心概念

- **意识上传**: 量子纠缠扫描，原子级精度复制大脑
- **机械飞升**: 从义肢到完全机械化的身体改造
- **AI层级**: 弱AI → 强AI → 超AI → 神AI
- **社会形态**: 纯碳联盟、赛博城邦、数字天国、AI自治区

## 创作建议

### 章节写作要点

1. **硬科幻基底**: 所有技术设定需有科学逻辑支撑
2. **人文关怀核心**: 技术是载体，人性是主题
3. **多线并进**: 个人命运与宏大历史交织
4. **哲学思辨**: 每章应有思想深度
5. **情感共鸣**: 不忘人类最基本的爱与恐惧

### 质量把控

- 定期使用 `story-consistency-analyzer` 检查章节一致性
- 保持人物行为符合已建立的设定
- 及时记录和追踪伏笔
- 确保时间线逻辑自洽
